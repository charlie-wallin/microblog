Thanks for your patience! Ive reviewed the routes.py file you uploaded. Here are several issues and potential improvements I found:

1. Incorrect Use of url_for() in Redirect

python
return redirect(url_for('/index'))

  • Problem: The argument to url_for() should be the function name (e.g. index), not the URL path.

  • Fix:

    python
    return redirect(url_for('index'))

2. Incorrect check_password() Call

python
if user is None or not user.check.password(form.password.data):

  • Problem: check.password is not a valid method. It should be user.check_password(...).

  • Fix:

    python
    if user is None or not user.check_password(form.password.data):

3. urlsplit(...).netlock Typo

python
if not next_page or urlsplit(next_page).netlock != '':

  • Problem: It should be netloc, not netlock.

  • Fix:

    python
    if not next_page or urlsplit(next_page).netloc != '':

4. Wrong Redirect Usage

python
return redirect('next_page')
return redirect(url_for('index'))

  • Problem: return redirect('next_page') is redirecting to a literal string 'next_page', not the variable.

  • Fix:

    python
    return redirect(next_page)

Also, the second return redirect(url_for('index')) is unreachable code because it comes right after the first return. Delete it.

5. Minor: Typo in "Sing in"

python
return render_template('login.html', title="Sing in", form=form)

  • Fix:

    python
    return render_template('login.html', title="Sign in", form=form)
